验证漏洞存在方法
    - 检查llama-cpp-python版本是否受影响
    - 验证模型加载时是否使用Jinja2模板渲染
    - 测试是否存在模板注入漏洞

漏洞利用方法
    - 该漏洞存在于llama-cpp-python的Llama类初始化过程中，当加载.gguf模型的元数据时，会解析chat template并使用无沙箱的jinja2.Environment进行渲染
    - 攻击者可以通过精心构造的payload在chat template中注入恶意Jinja2模板，导致服务器端模板注入(SSTI)
    - 由于Jinja2环境没有沙箱保护，攻击者可以执行任意Python代码，实现远程代码执行(RCE)
    - 漏洞利用需要构造包含恶意Jinja2模板的.gguf模型文件，当受害者加载该模型时触发代码执行

PoC
    - 构造恶意.gguf模型文件，在chat template元数据中注入Jinja2模板代码
    - 示例payload: `{{ config.__class__.__init__.__globals__['os'].popen('id').read() }}`
    - 当受害者使用llama-cpp-python加载该模型时，恶意代码将在服务器端执行
    - 攻击者可以获取系统命令执行权限，完全控制受影响的系统
