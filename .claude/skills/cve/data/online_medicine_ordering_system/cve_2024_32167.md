验证漏洞存在方法
    - 登录到Online Medicine Ordering System 1.0后台管理系统
    - 访问图片管理或设置功能模块
    - 尝试通过图片删除功能删除系统关键文件（如配置文件、数据库文件等）
    - 检查目标文件是否被成功删除

漏洞利用方法
    - 该漏洞存在于Online Medicine Ordering System 1.0的后端设置功能中
    - 系统的图片删除功能未对文件路径进行充分验证，导致攻击者可以删除任意文件
    - 攻击者可以通过构造恶意文件路径参数，删除系统关键文件，造成服务中断或数据丢失
    - PoC: 通过修改图片删除请求中的文件路径参数，指向系统重要文件（如config.php、database.php等），执行删除操作
