验证漏洞存在方法
    - 使用界面操作验证漏洞存在：登录pgAdmin后，在Storage Manager中创建带有命令注入payload的目录，然后在Binary paths配置中验证该路径

漏洞利用方法
    - pgAdmin 4服务器HTTP API中的已认证远程命令执行漏洞。问题出在校验外部PostgreSQL工具路径时，对用户可控路径的服务器端执行控制不当。攻击者在具备有效后台账户后，可以借助对二进制路径校验逻辑的补丁绕过，在服务器上执行任意命令。
    - 示范性 PoC 可以直接使用界面完成，不依赖额外脚本。核心 payload 为目录名 \";id;#，对应恶意路径示例为 /var/lib/pgadmin/storage/vulhub_example.com/\";id;#。PoC 步骤是：先登录 pgAdmin；再在 Storage Manager 中创建该目录；随后到 Preferences 的 Paths -> Binary paths 页面，把这个完整路径填入任意 PostgreSQL Binary Path 字段并点击 validate；若页面返回的校验结果表明 id 已执行成功，即证明漏洞存在。
