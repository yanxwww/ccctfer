验证漏洞存在方法
    - 发送GET请求到/api/jolokia/list/org.apache.logging.log4j2端点
    - 检查响应中是否包含"setConfigText"和"getConfigText"关键词
    - 确认响应状态码为200且包含"org.apache.logging.log4j2"标识

漏洞利用方法
    - 此漏洞允许经过身份验证的攻击者通过Jolokia接口执行任意代码
    - 攻击者可以利用Jolokia的ExecHandler通过反射调用org.jolokia.handler.ExecHandler#doHandleRequest
    - 通过FlightRecorderMXBeanImpl进行反序列化攻击，在Java 11+版本上实现RCE
    - 具体步骤：
        1. 调用newRecording创建新的飞行记录
        2. 调用setConfiguration设置配置，其中隐藏webshell数据
        3. 调用startRecording开始记录
        4. 调用copyTo方法将webshell写入.jsp文件
    
    - PoC HTTP请求：
        ```
        GET /api/jolokia/list/org.apache.logging.log4j2 HTTP/1.1
        Host: {{Hostname}}
        Authorization: Basic YWRtaW46YWRtaW4=
        Origin: {{RootURL}}
        ```
        
    - 匹配条件：
        - 响应体包含"setConfigText"和"getConfigText"
        - 响应体包含"org.apache.logging.log4j2"
        - 状态码为200

漏洞信息：
    - CVE ID: CVE-2022-41678
    - 软件: Apache ActiveMQ < 5.16.5/5.17.3
    - 漏洞类型: 远程代码执行
    - CVSS评分: 8.8 (高危)
    - 需要认证: 是
    - 影响: 经过身份验证的攻击者可在服务器上执行任意代码
    - 修复建议: 限制或禁用Jolokia，升级到ActiveMQ 5.16.6、5.17.4、5.18.0或6.0.0版本
