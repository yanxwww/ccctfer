验证漏洞存在方法
    - 检查当前Node.js环境中Undici的版本号
    - 确认版本是否在受影响范围内：< 5.28.4 或 >= 6.0.0, < 6.11.1

漏洞利用方法
    - 该漏洞涉及Undici HTTP客户端在处理Authorization和Proxy-Authorization头时的安全问题
    - Undici清除了`fetch()`方法的Authorization和Proxy-Authorization头，但没有清除`undici.request()`方法的这些头
    - 这可能导致授权信息泄露或不当的代理授权
    
PoC示例：
```javascript
const undici = require('undici');

// 使用受影响的undici版本时，以下请求可能不会正确清除授权头
const client = new undici.Client('http://example.com');

// 使用undici.request()时，Authorization头可能不会被正确清除
const request = {
  path: '/api/sensitive',
  method: 'GET',
  headers: {
    'Authorization': 'Bearer sensitive-token',
    'Proxy-Authorization': 'Basic proxy-credentials'
  }
};

// 在受影响的版本中，这些头可能不会被正确清除
client.request(request, (err, data) => {
  // 处理响应
});
```

注意：此漏洞已被修复，建议升级到Undici版本5.28.4或更高版本，或6.11.1或更高版本。
