验证漏洞存在方法
    - 检查 langchain-ai/langchain 版本是否受影响
    - 测试 XML 解析功能是否对递归实体引用有限制
    - 监控系统资源使用情况，观察是否存在异常的资源消耗

漏洞利用方法
    - 该漏洞允许攻击者通过构造包含多层嵌套实体的 XML 文档，触发 Billion Laughs Attack
    - 攻击者可以利用此漏洞使 XML 解析器消耗大量 CPU 和内存资源，导致拒绝服务 (DoS)
    - 漏洞利用的关键在于创建递归的实体引用，使解析器陷入无限扩展循环
