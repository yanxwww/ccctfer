验证漏洞存在方法
    - 尝试访问目标LibreChat实例的/api/files端点
    - 检查响应中是否包含文件操作相关的API功能
    - 验证目标系统是否运行LibreChat v0.7.5-rc2或受影响版本

漏洞利用方法
    - 该漏洞存在于LibreChat的/api/files端点，由于缺乏对文件路径的充分验证，攻击者可以通过路径遍历技术删除服务器上的任意文件
    - 攻击者可以构造恶意请求，使用../等路径遍历序列来访问和删除系统关键文件
    - 漏洞利用可能导致系统文件、用户数据或应用程序资源被删除，影响系统完整性和可用性
    
PoC示例：
```http
DELETE /api/files HTTP/1.1
Host: target-librechat-instance.com
Content-Type: application/json

{
  "filepath": "../../../../etc/passwd"
}
```

注意：这是一个概念验证示例，实际利用可能需要根据目标系统的具体配置进行调整。该漏洞的CVSS评分为9.1（CRITICAL），属于高危漏洞，可能对系统造成严重破坏。
