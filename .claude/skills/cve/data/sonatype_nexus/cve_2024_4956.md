验证漏洞存在方法
    - 发送包含路径遍历payload的HTTP GET请求到目标Nexus Repository Manager
    - 检查响应状态码是否为200
    - 检查响应头是否包含"application/octet-stream"
    - 检查响应体是否包含/etc/passwd文件内容（如"root:"等系统用户信息）

漏洞利用方法
    - 未经身份验证的攻击者可以通过路径遍历漏洞读取任意系统文件
    - 通过构造特殊的URL路径，利用多个斜杠和目录遍历序列绕过安全限制
    - PoC请求路径：GET /%2F%2F%2F%2F%2F%2F%2F%2F%2F%2F%2F%2F%2F%2F..%2F..%2F..%2F..%2F..%2F..%2F..%2Fetc%2Fpasswd
    - 匹配条件：状态码200、响应头包含"application/octet-stream"、响应体包含"root:"等系统用户信息
