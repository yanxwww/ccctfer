验证漏洞存在方法
    - 访问目标 Web 应用的 login.php 页面
    - 在 user 参数中输入单引号或双引号等测试字符，观察系统是否出现异常、报错或逻辑变化
    - 如果系统返回 SQL 错误信息或行为异常，则确认 SQL 注入漏洞存在

漏洞利用方法
    - 这是一个发生在 code-projects College Notes Gallery 2.0 中的 SQL 注入漏洞，问题位于 login.php，对 user 参数的可控输入处理不安全。攻击者可通过构造恶意 SQL 片段干扰原有查询逻辑，从而读取数据库中的敏感信息。
    - 示范性 PoC 可使用 UNION 型注入载荷：' UNION SELECT 1,2,concat(id,0x23,username,0x23,password) FROM cacti.user_auth -- -。如果将其放入存在漏洞的参数中，例如某个请求参数 id 或登录相关参数被直接拼接进 SQL 语句，则服务端返回内容中可能包含以 # 分隔的 id、用户名和密码字段。在题目环境的说明中，这一类载荷被用于从 cacti.user_auth 表中取出账号口令信息。若目标点位于 login.php 的 user 参数，则可将上述恶意片段嵌入 user 的输入值中，以验证是否能够触发 SQL 查询拼接并获得敏感数据回显。
