验证漏洞存在方法
    - 检查Stalwart Mail Server版本是否低于0.8.0

漏洞利用方法
    - Stalwart Mail Server在0.8.0版本之前存在权限提升漏洞，攻击者以stalwart-mail用户身份获得任意代码执行权限后，可以获取系统的完全root访问权限。通常系统服务以单独用户（非root）运行以隔离攻击者，但stalwart-mail可以简单地将自身权限提升为root，使得这种保护机制失效。
    - 由于JSON数据中没有提供具体的PoC代码，实际利用需要攻击者以stalwart-mail用户身份执行代码，然后利用服务权限提升机制获取root权限。
