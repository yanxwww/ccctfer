验证漏洞存在方法
    - 检查当前使用的jQuery版本是否在受影响范围内（>= 1.2, < 3.5.0）
    - 测试HTML片段在经过清理后，再传入jQuery的DOM操作方法（如.html()、.append()等）是否会导致恶意代码执行

漏洞利用方法
    - 这是一个jQuery的潜在跨站脚本漏洞。受影响版本在处理来自不可信来源的HTML片段时，即使这些HTML已经过清理，只要再传入jQuery的DOM操作方法，例如.html()、.append()等，仍然可能导致其中的恶意代码被执行。
    - 该问题在jQuery 3.5.0中得到修复。旁证文件还显示，Cacti在1.2.11中将jQuery升级到3.4.1以缓解相关XSS问题，并在1.2.13中继续通过供应链更新处理CVE-2020-11022和CVE-2020-11023，说明该漏洞会影响依赖旧版jQuery的上层应用。
    - 由于原始数据中未提供具体的PoC代码，建议参考官方修复方案：升级到jQuery 3.5.0或更高版本
