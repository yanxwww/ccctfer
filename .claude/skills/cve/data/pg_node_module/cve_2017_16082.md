验证漏洞存在方法
    - 检查当前使用的pg模块版本是否在受影响范围内
    - 尝试执行包含恶意列名的SQL查询，观察是否触发代码执行

漏洞利用方法
    - 此漏洞存在于pg模块中，当远程数据库或查询指定特制的列名时，会导致远程代码执行
    - 攻击者可以通过两种方式利用此漏洞：
        1. 执行不安全的、用户提供的SQL，其中包含恶意列名
        2. 连接到不受信任的数据库并执行返回结果的查询，其中任何列名都是恶意的
    - 漏洞的根本原因是CWE-94代码注入，攻击者可以注入恶意代码到列名中，当pg模块处理这些列名时会执行注入的代码
