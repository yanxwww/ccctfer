验证漏洞存在方法
    - 尝试访问目标服务器的 `POST /api/v1/images/delete` 端点
    - 检查是否能够通过该API接口删除文件

漏洞利用方法
    - 该漏洞存在于invoke-ai/invokeai版本v5.0.2的web API中，`POST /api/v1/images/delete`端点存在任意文件删除漏洞
    - 攻击者可以通过构造恶意请求，利用该漏洞删除服务器上的任意文件，包括关键系统文件如SSH密钥、SQLite数据库和配置文件
    - 这属于CWE-73（外部文件名或路径控制）漏洞类型，攻击者可以控制文件路径参数，导致任意文件删除
    
    - **PoC示例**:
      ```
      POST /api/v1/images/delete HTTP/1.1
      Host: target-server
      Content-Type: application/json
      
      {
        "file_path": "/etc/passwd"  # 或其他敏感文件路径
      }
      ```
      
      注意：实际利用可能需要根据具体的API参数格式进行调整，但核心是利用file_path参数的可控性来指定任意文件路径进行删除操作。
