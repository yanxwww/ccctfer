验证漏洞存在方法
    - 检查目标CouchDB实例是否在9100端口监听
    - 尝试使用默认Erlang cookie值"monster"连接到目标系统的9100端口
    - 如果连接成功并获得响应，说明目标可能受到此漏洞影响

漏洞利用方法
    - 此漏洞存在于Apache CouchDB 3.2.2之前的版本中，攻击者可以通过Erlang分发协议访问未正确保护的默认安装
    - 攻击者无需身份验证即可获得管理员权限，并执行任意命令
    - 漏洞利用步骤：
        1. 连接到目标CouchDB的9100端口（Erlang分发端口）
        2. 发送握手消息并接收挑战
        3. 使用默认cookie值"monster"计算MD5哈希响应
        4. 发送认证响应
        5. 发送命令执行payload
        6. 如果响应中包含"uid"、"gid"、"groups"等系统信息，说明命令执行成功
