验证漏洞存在方法
    - 向目标系统发送包含内部网络地址的URL（如 http://127.0.0.1:80 或 http://localhost:8080）
    - 观察系统是否尝试访问这些内部地址
    - 检查系统响应中是否包含内部服务的信息或错误

漏洞利用方法
    - 此漏洞是Server-Side Request Forgery (SSRF)类型，攻击者可以通过构造恶意URL，使服务器向内部网络发起请求
    - 攻击者可以利用此漏洞扫描内部网络、访问内部服务或进行其他内部网络攻击
    - 由于parse-url库在解析URL时存在缺陷，攻击者可以绕过安全限制，使服务器访问本应被阻止的内部地址
    
PoC示例：
    - 构造包含内部网络地址的URL，如：http://attacker.com?url=http://127.0.0.1:8080/admin
    - 发送到使用受影响parse-url库的目标系统
    - 观察服务器是否尝试访问内部管理接口
    - 如果成功，攻击者可以进一步利用此访问权限进行内部网络探测或攻击
