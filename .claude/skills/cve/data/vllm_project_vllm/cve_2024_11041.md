验证漏洞存在方法
    - 检查目标系统是否运行vllm v0.6.2版本
    - 确认目标系统暴露MessageQueue API接口
    - 尝试连接MessageQueue端口进行握手验证

漏洞利用方法
    - 该漏洞存在于vllm-project vllm v0.6.2版本的MessageQueue.dequeue() API函数中
    - 该函数使用pickle.loads直接解析接收到的socket数据，导致反序列化漏洞
    - 攻击者可以通过向MessageQueue发送恶意pickle payload来触发远程代码执行
    - 由于pickle.loads会执行反序列化对象中的任意代码，攻击者可以构造恶意payload执行系统命令
    
    - PoC概念：
      1. 构造恶意pickle payload，包含要执行的系统命令
      2. 连接到目标系统的MessageQueue API
      3. 发送恶意payload到MessageQueue.dequeue()接口
      4. 触发pickle.loads反序列化，执行恶意代码
      5. 验证命令执行结果或建立反向shell连接
