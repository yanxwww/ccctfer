验证漏洞存在方法
    - 检查是否使用了受影响的ImageClassificationDataset.from_csv() API
    - 验证输入的CSV文件中的URL是否来自不可信来源
    - 检查是否从不可信来源下载并提取tar.gz文件

漏洞利用方法
    - 该漏洞存在于dmlc/gluon-cv仓库的ImageClassificationDataset.from_csv() API中，版本0.10.0
    - 该函数从URL下载并提取tar.gz文件时没有进行适当的清理，使其容易受到TarSlip漏洞攻击
    - 攻击者可以通过制作恶意tar文件来利用此漏洞，当提取时，可以通过路径遍历或伪造的符号链接覆盖受害者系统上的文件
    - 漏洞本质是CWE-59：文件访问前链接解析不当，允许任意文件写入
    
PoC利用步骤：
1. 创建一个包含恶意URL的CSV文件
2. 恶意URL指向一个包含路径遍历payload的tar.gz文件
3. 当ImageClassificationDataset.from_csv()处理该CSV文件时，会下载并提取恶意tar文件
4. 提取过程中，恶意文件会通过路径遍历覆盖系统上的敏感文件

注意：由于安全考虑，具体的恶意tar文件构造细节和利用代码未在此处提供。在实际测试环境中，应使用可控的测试文件进行验证。
