    - 尝试访问open-webui的markdown转换端点（通常为/api/markdown或类似路径）
    - 发送一个简单的markdown测试payload，观察服务器响应时间是否异常
    - 检查端点是否无需认证即可访问

漏洞利用方法

    - 构造一个包含大量复杂嵌套结构或重复元素的markdown payload
    - 示例PoC：
```markdown

[重复大量链接和格式化内容]

**重复的格式化文本** *重复的格式化文本* ~~重复的格式化文本~~ `重复的格式化文本`

> 多层嵌套引用
>> 更深层嵌套
>>> 更更深层嵌套
>>>> 大量嵌套层...

1. 长列表项
2. 长列表项
3. 长列表项
...

[重复上述模式数百次]
```

注意：实际利用时需要根据目标服务器的具体实现构造更有效的payload，可能涉及特定的markdown语法组合来最大化资源消耗。
