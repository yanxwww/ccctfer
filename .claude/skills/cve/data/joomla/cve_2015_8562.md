验证漏洞存在方法
    - 发送包含恶意User-Agent头的HTTP请求，检查响应中是否包含"PHP Extension"和"PHP Version"等PHP信息

漏洞利用方法
    - Joomla! 1.5.x, 2.x, 和3.x（3.4.6之前版本）存在PHP对象注入漏洞，攻击者可以通过HTTP User-Agent头注入恶意序列化对象，执行任意PHP代码
    - 漏洞利用PoC：

漏洞影响
    - 攻击者可以执行任意PHP代码，导致服务器完全被控制
    - 可能导致数据泄露和系统完全沦陷

修复建议
    - 立即升级到Joomla 3.4.6或更高版本

    - CVE ID: CVE-2015-8562
