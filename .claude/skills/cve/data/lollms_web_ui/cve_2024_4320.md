验证漏洞存在方法
    - 检查目标lollms-webui应用是否存在/install_extension端点
    - 尝试发送POST请求到/install_extension端点，观察响应状态和内容
    - 验证应用版本是否为受影响的parisneo/lollms-webui最新版本

漏洞利用方法
    - 该漏洞存在于lollms-webui的/install_extension端点，由于对name参数处理不当导致本地文件包含(LFI)
    - 攻击者可以构造恶意的name参数，使服务器从任意位置加载并执行__init__.py文件
    - 通常利用上传目录中的discussions文件作为攻击向量，实现远程代码执行
    - 漏洞利用无需用户交互，当应用暴露在外部端点或以headless模式运行时风险最高
    - PoC：构造包含恶意路径的name参数，指向上传目录中的__init__.py文件，通过POST请求触发代码执行
